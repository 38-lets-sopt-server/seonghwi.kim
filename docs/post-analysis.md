# 게시글 기능 및 데이터 분석
에브리타임 화면 설계서에서 게시글을 서버 개발자의 관점에서 보면, 핵심 기능은 **게시글 작성, 전체 조회, 단건 조회, 수정, 삭제**입니다.
목록 화면에서는 여러 게시글을 보여줘야 하므로 전체 조회 기능이 필요하고, 상세 화면에서는 특정 게시글 하나를 보여줘야 하므로 단건 조회 기능이 필요합니다. 또한 글쓰기 화면에서는 게시글 작성 기능이 필요하며, 이후 게시글을 변경하거나 제거하기 위한 수정 및 삭제 기능도 필요합니다.

추가적으로 댓글/대댓글, 좋아요, 스크랩, 알림 기능도 생각할 수 있지만, 이번 1주차 과제의 중심은 게시글 CRUD이므로, 우선 게시글 기능을 기준으로 정리하겠습니다.

## 1. 저장해야 하는 데이터

게시글 기능을 위해 기본적으로 다음 데이터가 필요합니다.

- 게시글 ID
- 게시글 카테고리
- 제목
- 본문 내용
- 작성자 정보
- 익명 여부
- 작성 시각
- 수정 시각

추가로 댓글 수, 좋아요 수, 스크랩 수 같은 정보도 필요할 수 있습니다.

## 2. 요청해야 하는 데이터

클라이언트는 기능별로 다음과 같은 데이터를 요청해야 합니다.

- 게시글 작성: 제목, 본문 내용, 작성자 정보, 익명 여부
- 게시글 전체 조회: 게시판 카테고리 등 조회 조건
- 게시글 단건 조회: 게시글 ID
- 게시글 수정: 게시글 ID, 새 제목, 새 본문 내용
- 게시글 삭제: 게시글 ID

## 3. 응답해야 하는 데이터

서버는 요청에 따라 다음과 같은 데이터를 응답해야 합니다.

- 게시글 작성 응답: 생성된 게시글 ID, 성공 메시지
- 게시글 전체 조회 응답: 게시글 목록
- 게시글 단건 조회 응답: 특정 게시글 상세 정보
- 게시글 수정 응답: 수정 성공 메시지 또는 수정된 게시글 정보
- 게시글 삭제 응답: 삭제 성공 메시지

## 4. 검증해야 하는 조건

게시글 기능에서 서버가 검증해야 하는 조건은 다음과 같습니다.

- 제목은 비어 있으면 안 된다.
- 제목은 최대 글자 수 제한을 만족해야 한다.
- 본문 내용은 비어 있으면 안 된다.
- 존재하지 않는 게시글 ID로 조회, 수정, 삭제할 수 없어야 한다.
- 수정 시에도 제목과 본문 내용은 유효해야 한다.
- 수정 / 삭제의 권한을 확인해야 한다.

## 5. 정리

서버 개발자는 화면설계서를 보고 **어떤 데이터를 저장해야 하는지**, **클라이언트가 무엇을 요청하는지**, **서버가 어떤 데이터를 응답해야 하는지**, **어떤 조건을 검증해야 하는지**를 먼저 파악해야 합니다.  
이를 바탕으로 게시글 CRUD를 구현할 수 있습니다.
